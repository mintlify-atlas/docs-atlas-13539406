---
title: "Quickstart: install and run your first Oiyo project"
description: "Install Oiyo into a UniApp project, configure the Vite plugin, write your first page with definePageMeta, and run the dev server in five steps."
---

Oiyo wraps a UniApp project with auto-routing, a layout system, root context, and navigation guardsŌĆöall activated through a single Vite plugin. This guide walks you from an empty UniApp scaffold to a running Oiyo application. The full process takes less than five minutes on a fresh machine.

<Steps>
  <Step title="Prerequisites">
    Before installing Oiyo, make sure your environment meets the required versions.

    | Requirement | Version |
    | ----------- | ------- |
    | Node.js | `>=18.0.0 <22` |
    | pnpm | `>=8` |

    Oiyo requires an existing UniApp project as its base. If you don't have one, create a scaffold using the official UniApp CLI or the HBuilderX project wizard before continuing.

    <Info>
      Oiyo's `package.json` pins `node` to `>=18.0.0<22` and `pnpm` to `>=8`. Using a version outside these ranges may produce install errors or unexpected runtime behavior.
    </Info>
  </Step>

  <Step title="Install the package">
    Add `@skiyee/oiyo` as a dependency in your project root.

    <CodeGroup>

    ```bash pnpm
    pnpm add @skiyee/oiyo
    ```

    ```bash npm
    npm install @skiyee/oiyo
    ```

    ```bash yarn
    yarn add @skiyee/oiyo
    ```

    </CodeGroup>

    After installation, the `postinstall` script runs `oiyo prepare` automatically. This generates the initial `src/pages.json` manifest from any pages already in `src/pages/`. You should see the file created or updated in your project after `pnpm add` completes.

    <Note>
      Do not edit `src/pages.json` manually. Oiyo owns this file. Any changes you make will be overwritten the next time `oiyo prepare` runs.
    </Note>
  </Step>

  <Step title="Register the Vite plugin">
    Open `vite.config.ts` and register `OiyoPlugin`. If you are using UnoCSS (recommended), add `UnoPlugin` before it.

    ```typescript vite.config.ts
    import OiyoPlugin from '@skiyee/oiyo/plugin'
    import UnoPlugin from 'uni-unocss'
    import { defineConfig } from 'vite'

    export default defineConfig({
      plugins: [
        UnoPlugin(),
        OiyoPlugin(),
      ],
    })
    ```

    Plugin order matters. `UnoPlugin` must come before `OiyoPlugin` so that UnoCSS class scanning runs before Oiyo's route and layout transforms.

    <Tip>
      If you are not using UnoCSS, omit `UnoPlugin` and the `uni-unocss` import. Oiyo works without itŌĆöjust remove the `import 'uno.css'` line from `main.ts` in the next step.
    </Tip>
  </Step>

  <Step title="Set up the app entry point">
    Replace the contents of `src/main.ts` with the Oiyo-compatible entry. It uses `createSSRApp` from `@skiyee/oiyo/runtime` instead of the standard Vue import.

    ```typescript src/main.ts
    import { createSSRApp } from '@skiyee/oiyo/runtime'
    import App from './App.vue'
    import 'uno.css'

    export function createApp() {
      const app = createSSRApp(App)
      return {
        app,
      }
    }
    ```

    `createSSRApp` is Oiyo's re-export of Vue's SSR-compatible app factory. UniApp requires SSR mode for all platforms, including H5 and mini-programsŌĆödo not switch this to `createApp`.
  </Step>

  <Step title="Create your first page">
    Add a page file at `src/pages/home/index.vue`. Use `definePageMeta` to declare the layout and navigation bar title. Import reactive utilities from `@skiyee/oiyo/runtime`.

    ```vue src/pages/home/index.vue
    <script setup>
    import { onLoad, ref } from '@skiyee/oiyo/runtime'

    definePageMeta({
      layout: 'tabbar',
      style: {
        navigationBarTitleText: 'ķ”¢ķĪĄ - Oiyo',
      },
    })

    onLoad(() => {
      console.log('Page Load')
    })

    const title = ref('Hello')

    function jump() {
      uni.navigateTo({ url: '/pages/my/index' })
    }
    </script>

    <template>
      <view>
        <text>{{ title }} Home Page</text>
        <button type="primary" @click="jump">Go to My Page</button>
      </view>
    </template>
    ```

    The `layout: 'tabbar'` value tells Oiyo to wrap this page inside `src/layouts/tabbar.vue` at render time. Pages that omit `layout` fall back to `src/layouts/default.vue`. `definePageMeta` is a compiler macroŌĆöit has no runtime cost and is erased during the build.

    <Warning>
      The page file path determines the route path. `src/pages/home/index.vue` becomes the route `/pages/home/index`. Keep the directory structure intentionalŌĆörenaming a page file changes its URL.
    </Warning>
  </Step>

  <Step title="Run the development server">
    Start the Oiyo development server with the `oiyo dev` command.

    ```bash
    oiyo dev
    ```

    Oiyo watches `src/pages/` for new or removed files and updates `pages.json` automatically during development. You do not need to restart the server when adding pages.
  </Step>
</Steps>

<Note>
  Before exploring further, read the [project structure](/project-structure) page to understand the full directory layout Oiyo expectsŌĆöincluding where layouts live, how `App.vue` acts as the layout host, and why `pages.json` is generated rather than authored.
</Note>
