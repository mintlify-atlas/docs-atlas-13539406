Standard UniApp requires every page to be listed manually in `pages.json` before it can be navigated to. Add a new page component but forget to update `pages.json` and the route simply does not exist. In projects with many pages this becomes a maintenance burden and a common source of hard-to-diagnose bugs. Oiyo eliminates the problem entirely: `OiyoPlugin` scans `src/pages/` at build time and writes `pages.json` automatically from the files it finds and the metadata declared inside them.

## How Oiyo generates pages.json

When you run `oiyo dev` or `oiyo build`, `OiyoPlugin` traverses the `src/pages/` directory tree. Every `index.vue` it finds becomes a route. The plugin then reads `definePageMeta` from each file to extract style and layout options, and writes the result to `pages.json`.

## File-to-route mapping

Route paths mirror the directory structure under `src/pages/`:

| Source file | Route path |
|---|---|
| `src/pages/home/index.vue` | `pages/home/index` |
| `src/pages/my/index.vue` | `pages/my/index` |
| `src/pages/user/profile/index.vue` | `pages/user/profile/index` |

Nesting is unlimited. Each additional folder segment becomes an additional path segment.

## Generated pages.json

The `pages.json` in the Oiyo starter reflects both registered routes and the metadata each page declared via `definePageMeta`:

```json src/pages.json
{
  "pages": [
    {
      "path": "pages/home/index",
      "layout": "tabbar",
      "style": {
        "navigationBarTitleText": "é¦–é¡µ - Oiyo"
```

The `layout` field comes from `definePageMeta({ layout: 'tabbar' })` in `src/pages/home/index.vue`. Pages that omit `layout` in their `definePageMeta` call (like `src/pages/my/index.vue`) produce entries without a `layout` field and fall back to the `default` layout at runtime.

<Warning>
Do not edit `pages.json` by hand. It is generated by Oiyo on every `oiyo dev` and `oiyo build` run. Manual changes will be overwritten.
</Warning>

## Navigating between pages

Use UniApp's standard navigation APIs. The route path is the same path that appears in `pages.json`. For example, `src/pages/home/index.vue` navigates to the "My" page using `uni.navigateTo`:

```vue src/pages/home/index.vue
<script setup>
import { onLoad, ref } from '@skiyee/oiyo/runtime'

definePageMeta({
  layout: 'tabbar',
  style: {
    navigationBarTitleText: 'é¦–é¡µ - Oiyo',
  },
})

function jump() {
  uni.navigateTo({ url: '/pages/my/index' })
}
</script>
```

Use `uni.switchTab` for pages that are part of a native tab bar, and `uni.navigateTo` for stack-based navigation.

## Adding a new page

<Steps>
  <Step title="Create the file">
    Add a new `index.vue` inside a folder under `src/pages/`:

    ```
    src/pages/user/profile/index.vue
    ```
  </Step>
  <Step title="Add definePageMeta">
    Declare style and layout options at the top of the script block:

    ```vue src/pages/user/profile/index.vue
    <script setup>
    definePageMeta({
      layout: 'default',
      style: {
        navigationBarTitleText: 'Profile',
      },
    })
    </script>
    ```
  </Step>
  <Step title="Run oiyo dev">
    Start or restart the dev server. Oiyo detects the new file and updates `pages.json` automatically. The route `/pages/user/profile/index` is now live.
  </Step>
</Steps>

<Note>
After installing a new package that adds pages or configuration files, run `oiyo prepare` to regenerate setup files before starting the dev server.
</Note>
