Oiyo's auto-routing system manages `pages.json` on your behalf. Each page entry is generated from the file's path and the `definePageMeta()` declaration inside it. You should never need to edit `pages.json` directly—doing so will cause your changes to be overwritten the next time `oiyo dev` or `oiyo prepare` runs.

<Warning>
  `pages.json` is overwritten by `oiyo prepare` and `oiyo dev`. Do not edit it manually. Define all page configuration inside `definePageMeta()` in the corresponding SFC.
</Warning>

---

## File structure

The following `pages.json` is generated from the two pages in the Oiyo starter—`pages/home/index.vue` (which declares `layout: 'tabbar'`) and `pages/my/index.vue` (which declares only a `style`):

```json src/pages.json
{
  "pages": [
    {
      "path": "pages/home/index",
      "layout": "tabbar",
      "style": {
        "navigationBarTitleText": "首页 - Oiyo"
      }
    },
    {
      "path": "pages/my/index",
      "style": {
        "navigationBarTitleText": "我的 - Oiyo"
      }
    }
  ]
}
```

---

## Field reference

### pages\[\]

Each element in the `pages` array corresponds to one SFC under `src/pages/`.

<ResponseField name="path" type="string" required>
  Route path derived from the file's location under `src/pages/`, without the `.vue` extension and without the `src/` prefix. For example, `src/pages/home/index.vue` becomes `pages/home/index`.
</ResponseField>

<ResponseField name="layout" type="string">
  Layout name for this page. Set via `definePageMeta({ layout: '...' })`. Oiyo uses this field to resolve the layout component from `src/layouts/` at runtime. This field is an Oiyo extension—it is not part of the standard UniApp `pages.json` schema.
</ResponseField>

<ResponseField name="style" type="object">
  UniApp page style configuration. Populated from the `style` key in `definePageMeta({ style: { ... } })`. Accepts any field documented in the UniApp `pages.json` `style` spec, such as `navigationBarTitleText`, `enablePullDownRefresh`, and `backgroundTextStyle`.
</ResponseField>

## How entries are generated

The source of truth for each entry is the `definePageMeta()` call in the page SFC:

```vue pages/home/index.vue
<script setup>
definePageMeta({
  layout: 'tabbar',
  style: {
    navigationBarTitleText: '首页 - Oiyo',
  },
})
</script>
```

OiyoPlugin reads this at build time and emits the corresponding `pages.json` entry. A page with no `layout` key—like `pages/my/index.vue`—produces an entry without a `layout` field and falls back to the default layout at runtime.

<Note>
  The `layout` field in `pages.json` is added by Oiyo and is not recognized by UniApp itself. Oiyo's runtime reads it to drive layout resolution via `OiyoLayout`. UniApp processes the remaining fields (`path`, `style`) as normal.
</Note>

## Adding a new page

You never touch `pages.json` directly. To add a page:

<Steps>
  <Step title="Create the SFC">
    Add a new `.vue` file under `src/pages/`, for example `src/pages/profile/index.vue`.
  </Step>
  <Step title="Call definePageMeta">
    Declare the page's layout and style inside the SFC.

    ```vue pages/profile/index.vue
    <script setup>
    definePageMeta({
      style: {
        navigationBarTitleText: 'Profile',
      },
    })
    </script>
    ```
  </Step>
  <Step title="Run oiyo dev or oiyo prepare">
    OiyoPlugin detects the new file and appends a new entry to `pages.json` automatically.
  </Step>
</Steps>
